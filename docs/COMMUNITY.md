# FORM · Beta Community Setup v0.1

> Як організувати простір для 200 beta-ліфтерів.
> Власники: founder + `process-keeper` + `clinical-safety` (модераційні правила).

---

## 1 · Філософія спільноти

FORM-спільнота — простір для **прогресу, а не порівняння**.

Три незмінних правила:
1. **Без порівняння тіл.** Немає «твій прогрес vs мій прогрес» у публічних каналах.
2. **Без компенсаторної мови.** «Відпрацювати читмілу» — не в нашому словнику.
3. **Питань не соромляться.** Кожен міг забути, що таке РПЕ. Це нормально.

> Ці правила — не бюрократія. Вони існують тому що ми будуємо продукт для людей,
> яким важливо залишатися здоровими. Клінічна безпека — частина продукту.

## 2 · Платформа

### Closed beta (Wave 0-2, ≤ 200 учасників) → **Slack**

Чому Slack, не Discord:
- Приватність: invite-only, не public join
- Менш гучний — люди уважніші до сигналів
- Легше вести structured feedback threads
- Founder-proximity відчувається природніше

### Post-launch (публічна спільнота, 200+ → ∞) → **Discord**

Перехід після App Store release. Discord handles scale краще + має більше fitness-community precedents.

## 4 · Ролі та permissions

| Роль | Хто | Що може |
|---|---|---|
| **Founder** | George | Все |
| **Moderator** | 2-3 довірених beta-users (Wave 0) | Pin, delete, kick, mute |
| **Beta Tester** | Всі invited | Писати в усі публічні канали |
| **Observer** | Советник/інвестор | Read-only, можуть написати Founder в DM |

### Як обрати модераторів (Wave 0)

З перших 25 hand-picked users. Критерії:
- Активні у `#general` і `#feedback` перші 3 дні
- Не токсичні у коментарях
- Ідеально — хтось з тренерським або coaching background
- Просиш особисто в DM, не публічно

## 5 · Welcome flow (Day 0)

### Автоматичне welcome-повідомлення (Slackbot)

Коли новий член приєднується до workspace, Slackbot відправляє в DM:

```
Привіт, [Name]! 👋

Ти — один із перших beta-тестерів FORM. Ми не кажемо "перших 200" щоб звучало круто —
просто це правда, і це означає, що твій голос тут реально важить.

Ось що тобі треба знати:

1. **Твоя мета** — тренуватися як зазвичай, але звертати увагу на те, що Victor каже
   і як FORM вписується в твій ритм.

2. **Канал для глюків** → #form-feedback (баги, UX-проблеми, "це дивно")
3. **Канал для Victor** → #victor-feedback (тон, поради, що звучало дивно)
4. **Канал для перемог** → #wins (PR, прогрес, хайлайти)

Я (George, founder) читаю кожне повідомлення. Питай будь-що — відповім особисто.

Правила спільноти: [посилання на #welcome]
```

### Перший пост від founder у `#welcome`

```
Вітаємо у FORM Beta.

Ти тут не як "учасник дослідження". Ти тут як людина, яка допомагає збудувати
тренера, якого сама хотіла б мати.

Кілька правил:

— Ділися чесно. Навіть якщо "Victor сказав дурницю і я ігнорував тиждень" —
  саме це і треба знати.
— Без порівняння тіл і результатів. Прогрес у кожного свій.
— Якщо щось тривожить тебе (в продукті або особисто) — пиши мені в DM напряму.

Пишу тут щодня. Не менеджер продукту — founder, який теж тренується.

— George
```

## 6 · Тижневий ритм

| День | Що відбувається |
|---|---|
| **Понеділок** | Founder пост у `#general`: "Що тренуємо цього тижня?" |
| **Середа** | Автонагадування у `#form-feedback`: "Середина тижня — будь-які баги?" |
| **П'ятниця** | Founder пост у `#wins`: "Хто закрив тиждень?" |
| **Неділя** | Founder DM-рандомний sampling: 3-5 users — "Як тиждень?" |

### Щотижневий founder-update (щоп'ятниці, `#announcements`)

Шаблон:
```
**Beta Update — Week [N]**

Що ми почули цього тижня:
• [топ-3 речі які повторювалися]

Що ми вже зафіксили для зміни:
• [конкретно — не "дякуємо за фідбек"]

Що залишається відкритим питанням:
• [чесно — не знаємо, думаємо]

Тиждень [N+1]:
• [на що фокусуємося]
```

## 7 · Збір feedback

### Структурований (раз на 2 тижні)

Короткий форм у Typeform (5 питань, ≤ 3 хвилини):

1. Скільки тренувань цього тижня з FORM? (0 / 1 / 2 / 3+)
2. Найбільша проблема, яку ти зустрів? (open text)
3. Що найбільше сподобалося? (open text)
4. Victor звучить як: занадто суворо / ок / занадто м'яко / незрозуміло (scale)
5. Чи порадив би FORM другу зараз? (1-10 NPS)

Форм надсилається через Slackbot у `#form-feedback` з нагадуванням.

### Неструктурований (continuous)

- `#victor-feedback` моніториться щодня — всі конкретні фрази Victor, які звучали дивно, зберігаються в окремому Notion-doc
- `#cv-quality` відео-репорти → нумеруються → пріоритизуються weekly
- `#form-feedback` bug/UX items → GitHubIssues або linear щотижня

### Founder calls (Day 28+)

З BETA_PLAYBOOK.md: 40-60 calls між D28-D60.

Скрипт дзвінка (30 хв):
- 5 хв: представлення, мета розмови
- 10 хв: "Розкажи як виглядає твій типовий тренувальний тиждень"
- 10 хв: "Де FORM вписався добре? Де тертя?"
- 5 хв: "Що б ти сказав другові про FORM за 30 секунд?"

Записи (з дозволу) → транскрипт → NVivo або ручне кодування.

## 8 · Модераційні guidelines

### Що модерується

| Тип | Дія |
|---|---|
| Порівняння тіл ("ти схуд швидше ніж я") | Видалити + приватне попередження |
| Компенсаторна мова ("відпрацюю це завтра") | М'яке перефразування від модератора |
| Токсична мотивація ("слабко") | Видалити + попередження |
| Медичні питання ("у мене болить коліно що робити") | Перенаправити в DM до founder; не давати медичних порад у каналі |
| Харчові обговорення (калорії, "що можна їсти") | Нейтральна відповідь: "Victor допомагає з програмуванням, для нутриції краще до спеціаліста" |
| Ознаки дистресу ("я зірвався вже другий тиждень") | DM від founder/модератора. Clinical note якщо серйозне. |

### Що НЕ модерується

- Чесна критика продукту — навіть жорстка
- Технічні баги і UX-скарги
- Питання про Victor навіть якщо незручні
- Обговорення тренінгу (обсяг, техніка, прогресія)

### Escalation path

```
Рядовий модератор → Founder (DM) → [якщо потреба] зовнішній психологічний ресурс
```

Якщо хтось пише щось, що вказує на розлад харчування або серйозний дистрес:
1. Не давати порад публічно
2. Написати в `#clinical-notes` (приватний)
3. Founder відповідає в DM — з теплотою, без діагнозів, з посиланням на ресурс (лінія довіри UA/EN)

## 9 · Beta → Public: план переходу

| Milestone | Що відбувається |
|---|---|
| App Store release (Q4 2026) | Анонс у Slack + запрошення в Discord |
| Discord відкривається | Slack beta-members отримують роль "OG Beta" у Discord |
| Slack deprecation | Через 30 днів після Discord launch, Slack архівується |
| Discord структура | Аналогічна Slack, але + публічні канали з приєднанням без invite |

### OG Beta привілеї в Discord

- Роль "Founding Member" з іконкою
- Пріоритетний доступ до нових фіч
- Окремий канал `#founding-members` (навіки)
- Згадка в Changelog / release notes (opt-in)

## 10 · Метрики здоров'я спільноти

Відстежуємо щотижня:

| Метрика | Ціль Wave 1 | Ціль Wave 2 |
|---|---|---|
| DAU / invited ratio | ≥ 40% | ≥ 35% |
| Messages sent / active member / week | ≥ 3 | ≥ 2 |
| `#form-feedback` posts / week | ≥ 10 | ≥ 15 |
| `#victor-feedback` posts / week | ≥ 5 | ≥ 8 |
| Moderation actions / week | ≤ 3 (ціль — культура не карає) | ≤ 5 |
| Founder calls booked (D28+) | 40 з 100 | 60 з 200 |
| NPS (bi-weekly pulse) | ≥ 40 | ≥ 45 |
